# Роли в системе

| Роль      | Назначение                                                                 | Где используется                                        |
|-----------|-----------------------------------------------------------------------------|----------------------------------------------------------|
| `admin`   | Получает доступ к расширенному UI (CRUD пользователей, админ-расписание).   | Фронтенд (App/Header/Login), бэкенд при авторизации.     |
| `artist`  | Артисты труппы. Отображаются на вкладке «Артисты театра».                   | `/api/Users?role=artist`, админ-панель артистов.         |
| `director`| Режиссёры театра. Вкладка «Режиссёры».                                      | `/api/Users?role=director`, админ-панель режиссёров.     |
| `worker`  | Сотрудники монтировочного цеха/технические специалисты.                     | `/api/Users?role=worker`, вкладка «Монтировочный цех».   |

Все управляемые роли (`artist`, `director`, `worker`) поддерживают единый CRUD-интерфейс.  
Администратор создаёт пользователя, указывая обязательные поля (логин, пароль, ФИО, стаж) и опционально загружая фотографию.  
Фотографии сохраняются на сервере в каталоге `Uploads`, а в базе хранится имя файла. Клиент получает абсолютный URL вида `/api/media/{fileName}`.
